# TBX ブートストラップ計画

> このファイルはTBX設計書の一部です。`blueprint.md`（VM・辞書のアーキテクチャ）および `blueprint-language.md`（コア言語仕様）と合わせて参照してください。

### ブートストラップ

* フェーズ 1：アウター・インタプリタ（外側）の構築
* フェーズ 2：実行状態状態管理フラグと DEF と END によるワード定義
* フェーズ 3：地の文の即時実行（インタプリタモードの完成）
* フェーズ 4：地の文のコンパイルとメインルーチン実行（フルプログラムモードの完成）

以下、各フェーズの詳細を説明する。

#### フェーズ1: アウター・インタプリタの構築

TBX のアウター・インタプリタは、テキスト行をステートメント単位で処理し、インナ・インタプリタ（`vm.run()`）を通じて実行する。本フェーズでは **実行モードのみ** を実装する（コンパイルモードはフェーズ2で導入する）。

**行の読み込みと分解:**

1. 入力を1行ずつ読む。
2. 行の先頭トークン（識別子）を **ステートメント名** として取り出す。行番号が付く場合はスキップする。
3. 残りのテキストを **引数式** として扱う（空であってもよい）。

**ステートメントの実行:**

ステートメントを1つ実行するたびに、以下の命令列を一時バッファに構築し `vm.run()` で即実行する。

```
Xt(LIT_MARKER)                          // データスタック上にステートメント境界sentinelを積む
[引数式の命令列]                         // SYAパーサーが生成; カンマで複数引数が積まれる
Xt(CALL), Xt(stmt), Int(arity), Int(0)  // ステートメントワードをCALL (4セル形式)
Xt(DROP_TO_MARKER)                      // sentinelまでスタックを巻き戻す
Xt(EXIT)                                // 番哨フレームを検知してvm.run()を終了させる
```

- ステートメント名を辞書で検索し、見つからなければエラー。
- `arity`（引数の数）はステートメントのシグネチャから確定する（例: `PUTDEC` は1、`LET` は2）。
- `Int(0)` は `local_count`（ステートメント呼び出し側はローカル変数を確保しない）。
- `vm.run()` は `ReturnFrame::TopLevel` 番哨フレームを通じて `EXIT` 実行時に終了する。

#### フェーズ2: DEF/ENDによるワード定義

> Issue #116「ワード本体の終端判定機構（EXIT規約）が仕様に明記されていない」に基づく設計方針

システムに「今は実行中か、それとも定義中か」を示すフラグ（is_compiling）を持たせて `DEF` を「即時実行ワード（IMMEDIATE）」として実装する。


DEF/END のコンパイル動作詳細（DEF の挙動・EXIT 規約・RETURN のコンパイル・コンパイル進行中領域のライフサイクル・行番号ラベルのコンパイラ内部管理）については `blueprint-language.md` の「DEF/END — ワード定義のコンパイル動作」セクションを参照。

#### フェーズ3: 地の文の即時実行

> Issue #13「フェーズ3の「地の文」実行方式が未決定」に基づく設計方針

インタプリタモードを完成させる。DEF...ENDブロック外の「地の文（即時実行ステートメント）」をトークン単位でその場で即時実行する。

* DEF...ENDブロックを見つけたらフェーズ2の手順でワードを辞書に登録する。
* ブロック外の地の文は、1ステートメントずつフェーズ1で定めた命令列パターン（`LIT_MARKER → 式 → CALL → DROP_TO_MARKER → EXIT`）を一時バッファに構築し、`vm.run()` で即実行する。
* この段階ではプログラム全体を先読みしないため、後方で定義されたワードを地の文から呼ぶことはできない。
* REPL（対話型）モードはこのフェーズ3の動作と同等。

#### フェーズ4: 地の文のコンパイルとメインルーチン実行

フルプログラムモードを完成させる。ソースコードをシングルパスで処理し、地の文をメインルーチンとしてコンパイルしてから実行する。

* ソースをシングルパスで先頭から順に処理する（先読みなし）。
* DEFブロックは出現した時点で即座にコンパイルして辞書に登録する。後続のDEFはそれ以前に定義されたワードを使用できる。
* ブロック外の地の文は、フェーズ1で定めた命令列パターン（`LIT_MARKER → 式 → CALL → DROP_TO_MARKER`）を使い、メインルーチン本体（`dictionary[DP..]`）に順次コンパイルする（即実行はしない）。全ステートメントのコンパイル後、最後に `Xt(EXIT)` を追記してメインルーチンを確定し、実行する。
* **IMMEDIATEワードは地の文のコンパイル中でもその場で実行される。** これにより、語彙切り替えやコンパイル設定の変更を行うワードを専用構文なしに実現できる（PRGMAワードという独立した概念は不要）。ただし、IMMEDIATEワードはステートメントレベル（式の外側）でのみ使用できる。式評価中（関数呼び出し引数の評価中）に検出した場合はコンパイルエラーとする（詳細は「式内関数呼び出しのarity確定方式」参照）。
* 全行を処理し終えた後、メインルーチンのXtをVMに渡して実行する。
* 地の文はすべてのDEFより後に実行されるため、地の文で変更した実行時の状態はDEFのコンパイルには影響しない。コンパイル時の振る舞いを変えたい場合はIMMEDIATEワードを使う。
