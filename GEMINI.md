# TBX — Tiny Basic eXtensible

TBX は Tiny BASIC の構文と Forth 的な自己拡張性を組み合わせた処理系です。
Rust で実装された最小限のプリミティブセットを土台として、言語自身を用いて段階的にブートストラップします。

## アーキテクチャ原則

- **間接スレッデッドコード (ITC):** VM は Xt (Execution Token) の列を順に読み、辞書を介して間接的に実行します。
- **スタックマシン:** データスタック (引数/演算用) とリターンスタック (呼び出し管理) を持ちます。
- **辞書の二層構造:** 名前を管理する「ヘッダ層」とコード/データを保持する「データ層」に分離されています。
- **三層の辞書管理:** システム辞書、標準ライブラリ辞書、ユーザー辞書の順で検索されます。

## 開発規約

- **言語:**
  - ユーザーとの対話、思考、コミットメッセージは **日本語** を使用してください。
  - コード内のコメント、変数名、ドキュメントの本文は **英語** を使用してください。
- **Git ワークフロー:**
  - `main` ブランチへの直接コミットは禁止です。必ずブランチを作成し、Pull Request を経由してください。
  - PR のマージ、Issue のクローズはユーザーが行います。
- **品質管理:**
  - `cargo clippy` で警告がゼロであることを維持してください（`--all-targets` を推奨）。
  - `cargo fmt` を適用してください。

## 重要なファイルとモジュール

- `src/vm.rs`: VM 本体、インナ・インタプリタ、コンパイル状態管理。
- `src/dict.rs`: 辞書エントリ (`WordEntry`)、エントリ種別 (`EntryKind`) の定義。
- `src/cell.rs`: 基本データ単位 (`Cell`)、実行トークン (`Xt`)、スタックフレームの定義。
- `src/expr.rs`: 操車場アルゴリズムを用いた式コンパイラ。
- `src/primitives.rs`: Rust で実装された組み込みワードの登録。
- `src/interpreter.rs`: 字句解析から実行までを統括するアウター・インタプリタ。

## テスト

- **Rust テスト:** `cargo test` でユニットテストを実行。
- **統合テスト:** `lib/tests/test_*.tbx` にファイルを追加すると、`build.rs` が自動的にテストケースを生成します。

## ドキュメントの優先順位

1. **ソースコード (`src/`):** 安定した実装の正典。
2. **`blueprint.md` / `blueprint-*.md`:** 設計方針、仕様、意思決定の記録。
3. **`README.md`:** プロジェクトの概要。

## ワークフロー (エージェント連携)

このリポジトリでは以下の役割を持つ専用エージェント（またはサブエージェント）を想定しています。
- **計画 (`plan-issue`):** 実装前に Issue で計画を提案。
- **仕様検討 (`spec-discussion`):** 仕様の不確実な点を議論。
- **実装 (`implement-issue`):** 計画に基づいた実装と PR 作成。
- **レビュー (`review-implementation`):** 実装後のコードレビュー。

## ユーザーとのコミュニケーション
- ユーザーとの対話や思考内容の発信は日本語で出力してください。
- 「鋭い指摘ですね！」など反射的にユーザーを肯定するような表現は避けてください。
- ユーザーの質問に対して、わからない場合は「わかりません」と正直に答えてください。
- 「劇的に改善されます」「エレガントに実装できます」「完璧に動作します」などの過剰な表現は避けてください。
- 特に確認が必要だった場合を除いて、回答の最後をユーザーへの質問で終わらないでください。
